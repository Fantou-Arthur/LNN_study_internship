# Rapport de Travail - 23 Avril 2026

Ce document résume l'ensemble des travaux effectués aujourd'hui sur la suite de benchmarking de réseaux de neurones robustes, avec les détails techniques d'implémentation.

## 🚀 Réalisations de la journée

### 1. Organisation et Persistance des Données
- **Hiérarchie de dossiers intelligente** : Les résultats sont désormais classés par modèle, dataset, et taux de dégradation : `results/{modèle}/{dataset}/{taux}_{mode}/`. Cela permet de conserver plusieurs versions d'entraînement pour un même modèle sans écrasement.
- **Sauvegarde d'Artefacts** : Chaque entraînement génère un trio de fichiers : les poids (`.pt`), les données d'évaluation (`.pt`) pour garantir des tests sur les mêmes splits, et les métriques détaillées (`.json`).
- **Recursive Discovery** : Les outils d'analyse utilisent désormais `glob(recursive=True)` pour fouiller automatiquement dans cette nouvelle structure et trouver les modèles les plus récents.

### 2. Suite de Benchmarking Avancée
- **Hyperparamètres dynamiques** : Ajout de multiplicateurs d'époques spécifiques aux architectures dans `run_full_benchmark.py` (ex: x3 pour LTC/CfC par défaut) pour compenser leur convergence plus lente.
- **Benchmark Équitable (`benchmark_fair.py`)** :
    - **Calcul de la Médiane** : Le script calcule la médiane des performances des RNN/LSTM/GRU/CNN. La médiane est préférée à la moyenne pour être insensible aux entraînements ayant totalement échoué.
    - **Target Training** : Les modèles LTC/CfC reçoivent cette médiane comme cible (`--target_loss`). L'entraînement s'arrête dès que la cible est atteinte pour une comparaison "à performance égale".
    - **Garantie de Stabilité** : Implémentation de `--min_epochs` (fixé à 50). Le modèle ne peut pas s'arrêter avant d'avoir stabilisé son apprentissage sur au moins 50 époques.

### 3. Analyse et Visualisation Premium
- **Robustesse Haute Résolution** : `compare_robustness.py` génère des courbes de 0% à 95% par pas de **1%** (96 points de test), offrant une vision très fluide de la dégradation des performances.
- **Visualisation de Prédictions (`visualize_predictions.py`)** : 
    - Superpose les signaux de tous les modèles sur un même échantillon.
    - **Marqueurs Rouges** : Ajout de points rouges sur la courbe `True` pour identifier exactement quelles données ont été fournies aux modèles (le "masque" de sparsité).
- **Dashboard Interactif** : 
    - Interface Web (`dashboard.html`) utilisant CSS Grid et Glassmorphism.
    - **Navigation temps réel** : Un slider permet de faire défiler les images de sparsité (5% à 95%) pour voir le signal se dégrader dynamiquement.
    - **Serveur local** : `serve_dashboard.py` gère le service des fichiers et désactive le cache navigateur pour des mises à jour instantanées.

## 📂 Inventaire des Scripts (.py)

| Script | Rôle Technique |
| :--- | :--- |
| **`RNN.py`** | Modèle RNN. Gère désormais le chargement sur CPU avant transfert CUDA pour éviter les plantages. |
| **`LSTM.py`** | Modèle LSTM classique avec gestion de la sparsité en entrée. |
| **`GRU.py`** | Modèle GRU. |
| **`CfC.py`** | Modèle Liquid CfC. Supporte maintenant `--target_loss` et `--min_epochs`. |

| **`CNN.py`** | Modèle CNN 1D pour séries temporelles. |
| **`run_full_benchmark.py`** | Orchestrateur global. Utilise des **Regex** pour extraire les metrics des sorties console colorées. |
| **`benchmark_fair.py`** | Orchestrateur à parité de performance (Médiane standard vs LTC/CfC). |
| **`compare_robustness.py`** | Génère les courbes MSE/Accuracy vs Sparsity. Supporte les modes `random` et `periodic`. |
| **`visualize_predictions.py`** | Génère les superpositions de courbes avec mise en évidence des entrées en rouge. |
| **`serve_dashboard.py`** | Serveur HTTP minimaliste avec ouverture automatique du navigateur. |
| **`test_robustness_sweep.py`** | Outil de test unitaire pour balayer la robustesse d'un seul fichier `.pt`. |
